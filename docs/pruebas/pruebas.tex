Una vez construido el sistema, es necesario comprobar que se comporta de acuerdo con los requisitos definidos en el \autoref{CAP:ANALISIS_DISENO}. Este capítulo recoge dos planos de validación complementarios. El primero cubre las pruebas automatizadas, tanto del \textit{backend} como del \textit{frontend}, que verifican el comportamiento correcto de cada componente de forma reproducible. El segundo recoge los resultados de una encuesta de experiencia de usuario orientada a validar si la aproximación socrática resulta útil y aceptable para los alumnos a los que va dirigida la herramienta.
