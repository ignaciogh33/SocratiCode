Con el objetivo de verificar la corrección del código y detectar regresiones de forma temprana, se desarrolló una suite de pruebas automatizadas que cubre las dos capas principales del sistema, el \textit{backend} Django y el \textit{frontend} Vue~3.

En cuanto a las pruebas del \textit{backend}, se implementaron con el marco integrado en Django (\texttt{django.test.TestCase} y \texttt{TransactionTestCase}), junto con el cliente \texttt{APIClient} de Django REST Framework. Para cubrir los flujos asíncronos del \textit{streaming} SSE se empleó \texttt{AsyncClient}, el cliente asíncrono nativo de Django, y la cobertura se midió con \texttt{coverage.py}. En total se ejecutan \textbf{105 tests} que verifican los flujos de autenticación, el ciclo de vida de las sesiones de chat, el \textit{streaming} de respuestas del LLM, la moderación dinámica y la integración con el motor de ejecución Piston. La suite supera satisfactoriamente la totalidad de los tests y alcanza una cobertura del \textbf{99\,\%} sobre el código del \textit{backend}.

De manera complementaria, se implementaron pruebas unitarias para el \textit{frontend}. Las pruebas unitarias de componentes y vistas se ejecutan en un entorno JSDOM mediante \textbf{Vitest}. Como herramientas de apoyo se utilizaron \texttt{@vue/test-utils} para el montaje e interacción con componentes Vue, y \texttt{@pinia/testing} para inyectar estados de \textit{store} controlados en cada test. Los servicios HTTP se aislaron con \textit{mocks} de módulo gestionados por la API nativa de Vitest. La cobertura abarca los \textit{stores} de Pinia, los servicios de comunicación con la API, los guardias del \textit{router} y el conjunto de vistas y componentes de la interfaz. En total se ejecutan \textbf{345 tests}, todos satisfactorios, con una cobertura de sentencias del \textbf{91,6\,\%} global, mientras que las vistas alcanzan el \textbf{93\,\%}, el \textit{sidebar} el \textbf{96\,\%} y los componentes de chat el \textbf{96,7\,\%}. El componente \texttt{CodeEditor} se excluyó deliberadamente de las pruebas unitarias por su dependencia directa con Monaco~Editor, cuya validación requiere un entorno de navegador real.
