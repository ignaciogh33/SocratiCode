La transformación de un modelo de lenguaje de propósito general en un tutor socrático y restrictivo requiere de una disciplina técnica rigurosa conocida como Ingeniería de Prompts. En el contexto educativo, esto implica estructurar semánticamente las instrucciones base (\textit{system prompts}) para alinear de forma determinista el comportamiento de la IA con objetivos pedagógicos concretos, utilizando técnicas como la dialéctica, la mayéutica y el razonamiento contrafactual \cite{chang2023prompting}. Para lograr este alineamiento, el sistema debe ser instruido no solo asumiendo un rol filosófico, sino integrando reglas absolutas sobre lo que no debe generar. La literatura reciente destaca que los prompts más efectivos para la tutorización de programación emplean metodologías de evaluación y cuestionamiento jerárquico, donde el modelo primero evalúa el estado del código del alumno en su contexto interno, identifica el error lógico, y posteriormente formula una única pregunta socrática o pista heurística dirigida a la brecha de conocimiento detectada, garantizando así un andamiaje adaptativo \cite{kargupta2024instruct}.

Lograr que un modelo de lenguaje convencional funcione como un tutor socrático que sepa limitarse depende de un diseño de prompts muy cuidado. No basta con darle un papel a la IA sino que hay que estructurar sus instrucciones para que use métodos como la dialéctica o la mayéutica \cite{chang2023prompting}.
