  % Fila 1: ID y Actor
  \rowcolor{ulepsbased}

    \textbf{Identificador:} CU3 & \textbf{Actor principal:} Administrador
  \end{tabularx} \\ \hline

  % Fila 2: Nombre
  \textbf{Nombre:} Configuración global del sistema (patrón Singleton) \\ \hline

  % Fila 3: Descripción
  \rowcolor{ulepsbased}
  \textbf{Descripción:} \newline 
  El administrador accede al panel de Django Admin para configurar los parámetros globales del sistema: modelo LLM del tutor, modelo LLM de moderación y modo de moderación. La configuración se almacena en una única fila de la tabla \texttt{SystemConfig} (patrón Singleton). \\ \hline

  % Fila 4: Precondiciones
  \textbf{Precondiciones:} \newline
  1. El administrador tiene acceso al panel de Django Admin (\texttt{/admin/}). \newline
  2. El usuario tiene permisos de staff (\texttt{is\_staff=True}). \\ \hline

  % Fila 5: Flujo principal
  \rowcolor{ulepsbased}
  \textbf{Flujo principal:} \newline
  1. El administrador accede a \texttt{http://localhost:8000/admin/} e inicia sesión con credenciales de staff.\newline
  2. El sistema muestra el panel de administración con la sección ``Configuración del sistema''.\newline
  3. El administrador hace clic en ``Configuración del sistema''. El sistema redirige automáticamente al formulario de edición de la única fila existente (\texttt{changelist\_view} $\rightarrow$ \texttt{redirect(pk/change/)}).\newline
  4. El sistema muestra el formulario organizado en dos fieldsets:\newline
  --- \textbf{Moderación de contenido}: campo \texttt{moderation\_mode} (desplegable: Input + Output, Solo Input, Solo Output, Desactivada).\newline
  --- \textbf{Modelos LLM (Ollama)}: campos \texttt{llm\_model} y \texttt{moderation\_model} (texto libre, ej: \texttt{llama3.2}, \texttt{phi4-mini-reasoning}).\newline
  5. El administrador modifica los valores deseados y pulsa ``Guardar''.\newline
  6. El sistema, mediante el método \texttt{save()} sobrescrito, fuerza \texttt{self.pk = 1} antes de guardar, garantizando que solo exista una fila (Singleton). Persiste los cambios en PostgreSQL.\newline
  7. El sistema muestra confirmación de guardado exitoso. \\ \hline

  % Fila 6: Flujos alternativos
  \textbf{Flujos alternativos:} \newline
  \textbf{FA-01:} No existe configuración previa: El método de clase \texttt{SystemConfig.get()} (invocado por \texttt{changelist\_view}) ejecuta \texttt{get\_or\_create(pk=1)}, creando la fila con valores por defecto: \texttt{moderation\_mode='both'}, \texttt{llm\_model='llama3.2'}, \texttt{moderation\_model='llama3.2'}.\newline
  \textbf{FA-02:} Intento de crear una segunda fila: \texttt{has\_add\_permission()} devuelve \texttt{False} si ya existe una fila, ocultando el botón ``Añadir'' en el Admin.\newline
  \textbf{FA-03:} Intento de eliminar la configuración: \texttt{has\_delete\_permission()} devuelve siempre \texttt{False}, impidiendo el borrado de la configuración.\newline
  \textbf{FA-04:} Modelo LLM no disponible en Ollama: El sistema no valida la existencia del modelo en tiempo de guardado (validación diferida). El error se producirá en la siguiente petición de chat, donde \texttt{ollama.chat()} lanzará una excepción capturada por el flujo de error del CU1 (FA-05).\newline
  \textbf{FA-05:} Acceso por usuario sin permisos de staff: Django Admin deniega el acceso y redirige a la página de login del Admin. \\ \hline

  % Fila 7: Postcondiciones
  \rowcolor{ulepsbased}
  \textbf{Postcondiciones:} \newline
  1. La configuración queda persistida en la fila con \texttt{pk=1} de \texttt{SystemConfig}. \newline
  2. Las peticiones de chat subsiguientes utilizarán los nuevos valores de forma inmediata (lectura en cada petición mediante \texttt{SystemConfig.get()}). \\ \hline

  % Fila 8: Detalles implementación Singleton
  \textbf{Detalles de implementación (Patrón Singleton):} \newline
  -- \textbf{Forzar PK única:} \texttt{save()} sobrescrito: \texttt{self.pk = 1} antes de \texttt{super().save()}.\newline
  -- \textbf{Creación automática:} Método de clase \texttt{get()}: \texttt{cls.objects.get\_or\_create(pk=1)}.\newline
  -- \textbf{Impedir creación manual:} \texttt{has\_add\_permission()} $\rightarrow$ \texttt{not SystemConfig.objects.exists()}.\newline
  -- \textbf{Impedir eliminación:} \texttt{has\_delete\_permission()} $\rightarrow$ \texttt{False}.\newline
  -- \textbf{Redirección automática:} \texttt{changelist\_view()} $\rightarrow$ \texttt{redirect(f'\{obj.pk\}/change/')}. \\ \hline
