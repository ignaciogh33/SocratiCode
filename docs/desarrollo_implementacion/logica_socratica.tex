% El comportamiento socrático del tutor no reside en el modelo de lenguaje en sí, sino en las instrucciones que lo condicionan. El \textit{system prompt} de SocratiCode define la personalidad completa del agente: le asigna el rol de tutor cuya directiva principal es <<instruir, no asistir>>, le prohíbe de forma explícita entregar código corregido o soluciones directas y le ordena formular siempre una única pregunta socrática por mensaje para no abrumar al alumno. Internamente, el modelo evalúa en qué fase de comprensión se encuentra el estudiante (conceptual, lógica o sintáctica) para calibrar la dificultad de la pista, aunque este proceso de razonamiento jamás es visible en la respuesta final. El \textit{prompt} incluye también ejemplos de interacción real (\textit{few-shot prompting}) que ilustran el tono y la estrategia esperados, reforzando el comportamiento deseado sin necesidad de reentrenar el modelo.

% Además del \textit{system prompt} base, cada petición al LLM se enriquece con un segundo mensaje de sistema que inyecta de forma invisible el código fuente actual del editor del alumno y la última traza de ejecución. Esta información contextual permite al tutor razonar sobre la situación concreta del estudiante —referenciando líneas específicas de su código o el error de la consola— sin que el alumno tenga que copiar y pegar nada en el chat.
