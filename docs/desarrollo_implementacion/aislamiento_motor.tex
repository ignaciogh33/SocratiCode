% La integración del motor de ejecución se implementa en la aplicación \texttt{apps/compiler} del \textit{backend}. Cuando el alumno pulsa el botón de ejecutar en el editor, Vue envía una petición POST autenticada al \textit{endpoint} de Django con el código fuente, el lenguaje seleccionado y la entrada estándar opcional. Django construye el \textit{payload} conforme a la API de Piston y lo reenvía mediante una llamada HTTP síncrona a \texttt{/api/v2/execute} en la URL interna del servicio Piston (configurable vía la variable de entorno \texttt{PISTON\_URL}). Piston levanta el contenedor efímero con todas las restricciones de aislamiento descritas en la Sección \ref{SS:DI_TEC_MOTOR}, ejecuta el programa y devuelve una respuesta JSON con los campos \texttt{compile} y \texttt{run}, que contienen respectivamente los errores de compilación (si los hay) y la salida estándar de la ejecución.

% Django procesa esta respuesta para detectar los distintos estados posibles —ejecución correcta, error de compilación, \textit{timeout}, señal de terminación forzada o lenguaje no instalado— y construye una respuesta normalizada para el \textit{frontend}. El cliente recibe el resultado y lo muestra en el panel de salida integrado, junto con la última salida que también se incluye en el siguiente mensaje al tutor, cerrando así el ciclo en el que el LLM puede razonar sobre el comportamiento real del código del alumno.
