% La alternativa más sencilla para obtener la respuesta del LLM sería esperar a que Ollama termine de generar el texto completo y entonces enviarlo al cliente de una sola vez. Sin embargo, dado que las respuestas pueden tardar varios segundos en generarse, esta aproximación produce una experiencia de usuario deficiente: el alumno ve la pantalla congelada y no sabe si el sistema está funcionando. Para resolverlo, SocratiCode implementa un flujo de \textit{streaming} mediante Server-Sent Events (SSE): en lugar de esperar al texto completo, el \textit{backend} abre una conexión HTTP de larga duración y envía cada fragmento de texto (\textit{token}) en el mismo instante en que Ollama lo genera.

% Técnicamente, Django devuelve un \texttt{StreamingHttpResponse} con el tipo de contenido \texttt{text/event-stream}. Dentro de la función generadora asíncrona \texttt{event\_stream()}, el cliente de Ollama se instancia como \texttt{AsyncClient} y se itera sobre el \textit{stream} de \textit{chunks}; cada \textit{token} recibido se serializa como un evento SSE (\texttt{data: \{...\}\textbackslash n\textbackslash n}) y se envía de inmediato al \textit{frontend}. En el lado del cliente, Vue 3 abre una conexión \texttt{EventSource}, escucha los eventos entrantes y va concatenando los \textit{tokens} al mensaje en curso, actualizando la interfaz de forma reactiva conforme llegan. El resultado es la apariencia de escritura en tiempo real que el usuario percibe.
